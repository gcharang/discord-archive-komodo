[dscrd-hush] <dukeleto> any arbitrary data, any file extension. Hush is about freedom of communication
[dscrd-hush] <dukeleto> Supporting larger and larger files is more work, not just linearly, but new things need to be written to support files being larger than 1 memo or 1 transaction or 1 block, those are the big cut-offs. So those are the milestones of support
[dscrd-hush] <dukeleto> Files that fit in 1 memo, 1 transaction, 1 block, and then I will assess if files larger than 1 block should be supported
[dscrd-hush] <dukeleto> These sizes work out to being about: 512 bytes, 200KB and 4MB
[dscrd-hush] <dukeleto> So first will come support for files up to 512 bytes, then files up to 200KB (both sizes will cost just one transaction fee by default, but miners could very well process these transactions slower if additional fees are not given). Files up to 4MB will incur multiple transaction fees, because they require multiple transactions to store. So a 4MB file will cost about 20X the fees as a 200KB file, because a 200KB file can be stored in 1 transaction, while a 4MB file requires many more transactions to store the data
[dscrd-hush] <dukeleto> Miners are probably excited about File Upload feature, because it means the more people want to store files in Hush, the more transaction fees Hush miners will make. It creates a reason to make many Hush transactions, which makes Hush miners happy
