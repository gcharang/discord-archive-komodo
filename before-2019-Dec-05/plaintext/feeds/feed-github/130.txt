[jl777/komodo] Pull request opened: #1575 Add the ability to import different WIF formats
This adds the ability to import WIFs of different SECRET_KEY formats directly to komodod. This is useful in the event of airdrops. It adds an additional optional argument to the `importprivkey` rpc command, `secret_key`. This value can typically be found in `chainparams.cpp` of a coin.
For example, to import a BTC WIF, the command would be: 
`komodo-cli importprivkey <BTCWIF> "label" false 0 128`

128 being the SECRET_KEY value found here,  https://github.com/bitcoin/bitcoin/blob/12128087...
