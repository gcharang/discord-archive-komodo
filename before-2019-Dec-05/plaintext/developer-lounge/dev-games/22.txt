2. SDK's and more
========================================================
Primarily the focus are on Unity development, because well, I work in Unity. Yet the SDK's should be super duper portable, but all small things need to be set up. I've been using Odin Inspector to make a sexy tool that fetches news, updates, but also can install "modules", straight from git, but its fairly bare bones. To call it a SDK, takes a while to be tested, and I prefer to focus on quality and turnkey, instead of massive features. Never the less, the Unreal variant should match functionality. The high level layer 6AME (6AM Enhanced) has a dedicated dude working on the documentation and some optimization, writing out some tutorials, and I'll orchestrate something crazy so there is stuff to be distributed on the YouTubes, because that is what kids like these days. The SDK development will mostly be on-the-fly as I work on the games, finding the use cases for crypto condition contracts as I go. The SDK's will be **open source** for anyone to improve and tamper with; at this point there are like 400 things I like to add, so bear with me; probably up somewhere this week.
