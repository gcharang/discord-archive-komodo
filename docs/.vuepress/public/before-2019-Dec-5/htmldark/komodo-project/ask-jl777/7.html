Regarding the discussion that happened with @ProficieNt : Mostly the scalable asset chains are going to be used for utility/app tokens. The main function miners provide are validation(Checking the transactions are valid and making blocks), security based on decentralization(i.e., if one miner is bad(submitting invalid blocks), the rest of the miners won't accept his blocks), redundancy: some miners go offline - others can continue mining making the system resilient and large hash power (to protect from 51% attacks). 51% attacks are mainly double spend attacks - can be ignored for dPOW if the last notarizid block is considered as valid. As resulted from the above discussion, a miner with large hash power can only attack by censoring transactions. Can the other miners simply block and stop accepting his blocks? For other block-chains this won't be feasible because the large hash rate is required for protection, but with dPOW this is less of an issue? Another conclusion was that by splintering into a huge number of chains, we are breaking up POW and POS security both by breakung up hash power and stake/wealth , I don't know about POS, but can't all the asset chains be merge mined so that all of them get the combined hash power?
