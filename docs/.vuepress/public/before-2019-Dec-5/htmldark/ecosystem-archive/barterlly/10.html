about handling specific case, made minor changes to status, in case of an addy gen problem, depending on the side that didn't have its address generated, there will be a different status code : # BOTH EMPTY : 210 # SRC MISSING : 200 # DST MISSING : 205 # BOTH OK : 3 A specific script will then loop on 200 210 and 205 to check availability on a 5 min timeframe, as soon as the coin is back, it will try only the coin that is missing. if succeed : go to status 3 (BOTH OK) and back to the main path
