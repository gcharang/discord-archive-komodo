daemonfox 19-Nov-19 02:15 AM
not exactly
If you check the KMDLabs git there is a pos64staker repo
that repo has several commands to split your coins to many addresses and UTXOs.
Its a balance game... how much coin in each utxo, how many utcos per address compared to the rest of the stakers
All dictates your performance (edited)
The idea is... there are 64 SEGIDs, 0-63
Every address is tied to a single SEGID
the staker checks the next SEGID in order for a valid block solution, then moves to the next until someone has a valid utxo that meets age and a solution, then it stakes a block.
So, idea is, get a good amount of coin, and if you have enough, divide it among the SEGIDs into several UTXOs per SEGID
the repo explains how
if you dont have a good amount... its likely better to make a larger UTXO on a single address
you can evaluate the last staked SEGIDs totals per SEGID in the last X blocks by passing the daemon CLI the getlastsegidstakes X
So, the last ~24 hours would be...
komodo-cli -ac_name=LABS getlastsegidstakes 1440
