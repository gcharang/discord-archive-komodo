i think zack can always find something wrong with everything, the question is if is at an acceptable risk level. the biggest exposure that is see is that if notary nodes are evil, then they could mislead newly connecting nodes onto a false chain. however as long as users run make sure their node is in sync with the mainchain, then any false chain is meaningless, especially if bets are not made until after in sync. (edited)
the other issues is that if the source of data is hacked, then the data will be controllable. but my point is that these oracles are about what the online pricefeeds are returning, not their abstract price independent of a price source, so in a way if a price source is hacked, as long as the oracles are mining what the price source had, it is correct
from a practical point, having the chain not go off on forks due to data that is different on each node, this seems the most real issue. For over a week all 10 of the test nodes are in lock step. We havent pen tested it yet and likely there will be some issues found and fixed during that process, but as near as I can tell we are very close to having a trustless pricefeed that can be used to settle bets: http://159.69.45.70:8050/
