ca333 08-Jul-19 03:32 PM
This is the dev channel for the ecosystem dashboard project. The github repo for the core API is hosted at https://github.com/KomodoPlatform/komodo-ecosysboard/ - @Slyris is the dev and implementing this. The (codename) "ecoboard" project is split into 2 subprojects: A) the backend layer (above github repo) is a (golang) service that fetches ecosystem project related infos/data from various publicly accessible APIs - this includes blockexplorers [kmdexplorer.io, vrsc.kmdexplorer.io,...], coin market APIs [coinmarketcap, coingecko,...] and other 3rd party APIs. Furthermore this system persists a specific set of data locally (last price, blockchain statistic numbers, etc.) and serves a publicly accessible API so that other 3rd party projects can utilize it. The backend can also request realtime data from external APIs like current blockchain stats by forwarding a request to external endpoints. B) the frontend layer is a mobile ready (responsive) website based on dashboard templates. This website interacts with the backend layer from A) - e.g. users can view data (get) or send specific requests (post) like blockexplorer queries. (edited)

ca333 11-Jul-19 10:18 PM
you make a request block, if it is not a block I test if it is a transaction, if it is not a transaction I test if it's an address? - lets only forward the request to the AP IF the user-input is pre-validated and filtered as best as we can to prevent explorer API spam: If its a decimal number <= current blockheight its a block. If its a 27 - 34 chars long base58 encoded string (prefix byte 60dec / 3Chex resulting in all KMD addresses starting with R) we know its an address (komodod has a validateaddress call but you can also use any base58 lib to validate the userinput). And if its a 32 byte long HEX its likely a TX ID or blockhash - check for TX hash first since endusers are more likely going to look for TXs.
