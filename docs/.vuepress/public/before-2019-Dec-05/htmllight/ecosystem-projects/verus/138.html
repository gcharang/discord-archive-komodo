[dscrdvrsc] <Crupti> @zpajk There is a lot going on in the Verus hashing algo, but I think the most heavy innerloop happens in verusclmulwithoutreduction64alignedrepeat() , see https://github.com/VerusCoin/nheqminer/blob/master/nheqminer/crypto/verus_clhash.cpp for the code. There is a lot of haraka stuff happening too, but only one Haraka per inner hash and one call to verusclmulwithoutreduction64alignedrepeat() which is where most of the heavy CPU happens. They use a lot of Intel SIMD intrinic functions, which are basically integrated functionality inside newer (2012 and later usually) CPUs - these functions are super efficient on the CPU which is why CPUs are better than GPUs on Verus.
