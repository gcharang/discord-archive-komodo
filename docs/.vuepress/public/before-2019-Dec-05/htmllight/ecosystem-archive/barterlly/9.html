maxijb 30-Oct-18 05:50 AM
So, I've been thinking about the hardware requirements... we'll need to run 2 nodeJS processes, so that should require 2 cores, plus mysql that will do all the heavy work, so maybe 4 cores are a reasonable setup
About memory , a nodeJS app has a limit of 1.8gb of ram, that times 2 is 3.6 gb, plus mysql... I think we could be ok with 4GB of ram to start
Disk space, the biggest chunk will be ocupied by the database... how much can that grow depends on how much traffic we'll get... let's say we start with 2GB? Plus a 100 or 200mb of logs... then the source code will not take much space... probably 8 GB will be more than enough
Of course, the test environment will require much less equipment, probably we can live with 2 cores, 2gb memory, 2gb space
And the way that nodejs applications scale -as they are single threaded- is by adding more instances, so let's say than one node can handle 10k concurrent users (this is a benchmark that I saw online but we'll need to measure it on our app to have a clear picture), we should monitor our cpu usage and whenever that's consistently high, we could add more cores, and run more instances on those new CPUs
That's a conservative estimation I can make a priori, we'll know better once we start serving requests of course... not sure how easy it is to scale hardware in your infrastructure?
