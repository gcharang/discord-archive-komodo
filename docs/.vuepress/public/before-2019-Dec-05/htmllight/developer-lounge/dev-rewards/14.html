lukechilds 09-Feb-19 07:31 AM
This is incorrect:
While Ledger hardware wallets have supported storing Komodo for quite some time, previously it was not possible to claim the 5% Active User Reward, as it did not set a transaction LockTime which is used to calculate the rewards.
nLockTime is required on transactions for the outputs to accrue interest, however this wasn't the issue on the Ledger, transactions sent to the Ledger from elsewhere with nLockTime set would still accrue interest. The issue was that actual KMD app running on the Ledger could not sign reward claim transactions because the outputs are larger than the inputs. This breaks BTC consensus rules the Ledger device would throw an error and refuse to sign. This needed to be fixed in the KMD Ledger app (that runs on the device, not Ledger Live) before any software could get the Ledger to sign a reward claim transaction

lukechilds 28-Feb-19 03:15 AM
@PTYX it's kind of complicated to explain (edited)
Short version: Now UTXOs created by transactions originating from Ledger Live will accrue rewards
previously they wouldn't
Long version:
Previously if I sent you 100 KMD fro my Agama wallet to your Ledger Live wallet, Agama will set nLockTime so the UTXOs sitting in Ledger Live will accrue rewards. You can then use our Ledger reward claim app to claim those rewards. Everything is good. However, if I sent you that 100 KMD and you then sent 5 KMD to somebody, Ledger Live would use your 100 KMD UTXO and split it into 5 KMD to the recipient and 95 KMD back to you as change. However, it doen't set nLockTime. So now your 95 KMD balance is sitting in a change UTXO from a transaction with no nLockTime set so will no longer accrue rewards. This PR fixes that. It makes sure all KMD transactions created by Ledger Live have nLockTime set so you can make transactions and your change UTXOs will always accrue rewards. (edited)
It also means any recipients you pay from Ledger Live will also start earning rewards on their UTXOs
