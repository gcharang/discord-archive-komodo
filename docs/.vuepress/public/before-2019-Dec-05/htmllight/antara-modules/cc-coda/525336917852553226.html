jl777c 15-Oct-19 04:10 PM
so each side is a 1:1 chain to the other side, the coin emission needs to be balanced between block production on each side. this allows a coda tech based chain that has a built in 1:1 komodo protocol chain
one advantage is that it would allow all the existing exchanges to be able to immediately integrate it without any additional customizations, as the komodo protocol side is standard bitcoin rpc
and the people who are using the coin would migrate the coins over to the coda side. so the traditional side will also get all the antara modules available, and that would allow a dual-daemon chain to have literally thousands of different instantiations. you could make a stablecoin backed by KMD and able to be 1:1 migrated automatically to the coda side.
if the coin emissions side can be managed, a BTC pegged chain can also be created, that has the coda tech that people can use it with. i hope you can see the spectrum of possibilities, basically unlimited.
it could be the most powerful usecase is for massive scaling, so the coin emissions are done on the komodo side and then the actual high volume usage on the coda side. that means the blockchain side will stay constant regardless of the number of transactions
