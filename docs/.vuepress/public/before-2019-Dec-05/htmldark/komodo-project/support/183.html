Oszy 03-Nov-19 09:41 PM
No not really like what you wrote.
SegID's are on chain (THC chain has 64 segid's) (edited)
Your wallet, ideally, should have 64 transparent addresses, each assigned to a SegID. One transparent address can contain thousands of different size of UTXO's, but only belong/stake on one SegID. AFAIK block explorer doesn't show segid. You can get stats with getlastsegidstakes "blocks" (edited)
Chew on the code of this, might give you an idea on what's happening behind the scenes. (how to properly manage UTXO's for staking on a Pos64 chain) https://github.com/KMDLabs/pos64staker
KMDLabs/pos64staker
Automated staker for PoS assetchains. Contribute to KMDLabs/pos64staker development by creating an account on GitHub.
You want to fill up all SegID's as much UTXO's as you can, with as big UTXO's as you can.
So say you have 100K coins, you want to do 100.000 / 64 ( 1562 UTXO's on each SegID with size 1, if you want bigger UTXO's you increase size, decrease number of UTXO's) (edited)
if you only have 100k on segid 0 in one UTXO it will less likely stake than having that 100k split into 64. because there are 63 other segid's that can stake
Hope this is ELI5 enough
