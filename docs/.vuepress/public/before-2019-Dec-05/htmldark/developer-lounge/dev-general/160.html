zcash/zips
Zcash Improvement Proposals. Contribute to zcash/zips development by creating an account on GitHub.
The removal of ephemeralKey and encCiphertexts fields means that for JoinSplit descriptions in v5 transactions, it is not necessary for a wallet to perform trial decryption of those fields to detect incoming payments as described in [4] sections 4.18 and 4.16.2. The processing of nullifiers in the algorithm of section 4.18 is unaffected.
and
Removing the ephemeralKey and encCiphertexts fields reduces the size of a JoinSplit description in a v5 transaction from 1698 bytes to 464 bytes, and avoids the need to scan those fields for incoming payments.
