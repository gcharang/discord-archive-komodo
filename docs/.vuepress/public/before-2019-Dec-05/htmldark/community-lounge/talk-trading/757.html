grewalsatinder 09-Jun-19 05:51 PM
To not dependent TOO MUCH on dependencies and code your code as much possible as you can, and review the imported code thoroughly, and also in case you really do depend on the dependencies lock the version number of the dependency to NOT auto upgrade to latest without a review.
The best security practices MUST be followed by the devs who build the applications.
No matter which programming language one uses, there are practices which help. from C to any other modern language all are like getting reusable code from other places via libraries, how the devs manage the security and practice their coding of software is a big thing to keep things secure and isolated it out of vulnerabilities.
With node.js I just find it too much of work to review code of 100s of other unknown dependencies which has further of their own dependencies, and so on and on. So, the best is to code all the those things as much possibly as you can by your own self.
but since with package manager and modern languages allow to share even a small notify feature like code, many devs just pick the 3rd party code and re-use it.
just a small example, I can't emphasise more on just code the code even if it's small just by your own self and use it.
not to depend on too much on small things on 3rd party.
as lesser the software will need dependency on external sources at compile time the best it is.
