Even then it sucks to require the recipient to run an open-to-the-Internet server at all. And for goodness' sake, don't use the broken/centralized HTTPS system; the Bitcoin Core devs have been going to a lot of trouble trying to remove that garbage from Core. Currently I think that there's no way for the recipient to get the modified slate after running receive the first time (eg. if it gets lost), which is nuts.
