[artemii235/SuperNET] New comment on issue #340: Seed nodes: key-value HTTP fallback
One interesting idea here is to use an Amazon S3 buckets as the seed nodes. That way any user can easily rent a new bucket and adjust the mm2 configuration of the exchanging parties to use it. And the S3 buckets might be more tamper-free than a seed node provided by an unknown party. The marketing goal here is to get rid of custom seed nodes, improving the transparency and trust-worthiness of the mm2 network.
