About versioning it's little bit not clear, i'm agree. Internally we have few different version numbers, but common is: - 3.0.0.0 (CLIENT_VERSION_MAJOR.CLIENT_VERSION_MINOR.CLIENT_VERSION_REVISION.CLIENT_VERSION_BUILD) - this is daemon version, that equals to v3.0.0-beta1. And equals "version": 3000000 from getinfo output. - 0.5.0 - is a KMD version ("KMDversion": "0.5.0"). - v3.0.0-beta1-9979ca2 - GitHub Tag of current release. Earlier (from times when Komodo-Qt was available only for Windows) we used different tag versioning, with -g%VER%-%BUILD% at the end, now it will be deprecated. (edited)
