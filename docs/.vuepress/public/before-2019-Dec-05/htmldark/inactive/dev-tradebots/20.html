grewalsatinder 29-Jul-18 06:50 AM
in BarterDEX you don't actually need ETH/ERC20 synced to local machine. I'm sure many machines today will just fill up the whole HDD with ETH blockchain. in BarterDEX the marketmaker uses a public ETH Parity Node, which acts like a SPV node for ETH/ERC20. but in marketmaker the API is set to use the same naitve API command, so just for that the GUI is using the native switch when activating the ETH/ERC20 coin. In short just activate ETH/ERC20 coin in GUI with 'Native' switch/option selected and you Don't need to sync the full blockchain of ETH/ERC20 on your machine.
in case of any confusion. @Hermes
