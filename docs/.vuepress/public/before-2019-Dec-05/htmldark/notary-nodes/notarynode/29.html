How dPoWConf works? dpowconf: 0, 1, n values in gettxout, listunspent or getblock call first stage
"confirmations": 0,
2nd stage
"confirmations": 1, "rawconfirmations": 1,
then after 5 KMD block confirm
"confirmations": 1, "rawconfirmations": 5,
after 12 KMD block cnfirm when a BTC notarization happen
"confirmations": 12, "rawconfirmations": 12,
exchanges can use, when confirmations is not 0 or 1 but n to allow withdraw or credit deposited funds. that means those txs are dpowed and secure. (edited)
