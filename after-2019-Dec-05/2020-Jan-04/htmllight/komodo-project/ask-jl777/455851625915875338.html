How to read DEXRDO prints:
jl777cLast Wednesday at 8:02 PM this print has 3 parts DEXpurge.145 for t.xxx -> n.0 that means for timeslot 145 at the unixtimestamp, it purged 0 packets this makes sense as you need to run 59 minutes before anything will be purged.the rightmost part of L.8 A.1 coll.0 is the performance stats of the internal hashtable for recent requests. L is the number of lookups, A.1 is the number of additions to the hashtable and coll is the number of collisions each collision will likely create a bit of redundant data transmission, that is why this is being tracked. my results show about a 0.1% collision rate, which is small enough to not matter as the goal is less than 20% redundant packets now the middle part is what has the most important data total.1 is the total number of packets that are in the RAM (sum of all timeslots) the e59... is a combination of all the shorthashes (least 32 bits of actual hash XOR), given N packets, if it generates the same combination of shorthashes, odds are very good that the data is 100% in sync R.1 S.0 A.1 duplicates.0 R.1 is the number of packets received S.0 is the number of packets sent A.1 is the number of packets added to the RAM. until purging starts, this should match the earlier total
